# Проект "Интеллектуальная система экспертной проверки письменных и устных ответов по английскому языку"

## Описание
Система в формате чат-бота, которая принимает письменные и устные ответы на задания экзаменов по английскому языку, анализирует их и выдает оценку и развернутый фидбек на основе официальных экзаменационных критериев. 

Цель - приблизить автоматическую проверку к уровню экспертной оценки.

## Шаги работы:
1. В первой итерации будем решать задачу text-to-text - даем модели эссе, она дает оценку с комментариями  
2. Начнем с prompting:  
   Полученный результат будем считать за baseline, который будем пытаться перебить с помощью  
a) Улучшения промпта (chain of thoughts, детализация и тд).  
b) Более мощной модели по числу параметров.  
с) Другой модели.  
d) Перехода в fine-tuning  
  

## Prompting
## 1. Формирование baseline
1. Загрузим легкую модель Qwen/Qwen2.5-3B-Instruct из hugging face и на простом промпте измерим качество.  

Полученный результат будем считать за baseline, который будем пытаться перебить с помощью  
a) Улучшения промпта (chain of thoughts, детализация и тд).  
b) Более мощной модели по числу параметров.  
с) Другой модели.  
d) Перехода в fine-tuning.  
  
2. Специфика данных:

Эссе оценивается по 4-м критериям, итоговая оценка это просто среднее из них. Поэтому мы хотим от модели попадания в каждый из критериев, помимо общей оценки.

3. Выбранные метрики качества:

MAE - покажет в баллах на сколько ошибаемся в предсказании, что очень наглядно. Будем в идеале стремиться к тому, чтобы модель ошибалась менее, чем на 0.5 балла  
RMSE - учитываем выбросы. Считаем, что дать сильно неправильную оценку за эссе это критически плохо  
MBE - модель в реднем завышает или занижает предсказания? Будем стремиться к тому, чтобы модель минимально завышала/занижала оценки.  

4. Формирование пайплайна
a) Подача промпта c примерами работ из тестового датасета
b) Работа модели на основе промпта на случайной подвыборке из тестового датасета
с) Оценка качества модели
Этот же код будем использовать при тесте других моделей/ промптов

## 2. Написание сильного промпта 
Ограничения, с которыми столкнулись при промптинге:
- поиск трейд-офа скорости генерации, количества токенов и развернутости комментария

Бейзлайн строился с лимитом в 600 токенов, что не позволяло добавить CoT и детализированный комментарий
Параметры и механизмы, которые протестировали:
- количество токенов(600, 2000)
- CoT
- Контрастные примеры
- Экспертные дополнения

Наилучший результат остался в базовом промпте и его дополнении контрастными примерами. Эксперименты продолжим, но пока лучший набор метрик такой:

| :--------------------------- | :----- | :----- | :----- | :---------- | :----- |
| Baseline                     | 1.070  | 1.219  | -0.070 | 0.165       | 0.019  |
| Контрастный промптинг        | 1.290  | 1.663  | 0.670  | 0.215       | 0.140  |


## 3. Тест других моделей 
1. Загрузили различные модели для сравнения. Финально остановили выбор на следующих моделях, поскольку они имеют большое количество скачиваний на Hugging Face, не требуют дополнительных аутентификаций и находятся в топе:
- microsoft/Phi-3-mini-128k-instruct 
- unsloth/Llama-3.2-3B-Instruct-bnb-4bit (Llama)
- Qwen/Qwen2.5-3B-Instruct (более легкая модель Qwen)
- Qwen/Qwen2.5-7B-Instruct (более тяжелая модель Qwen, но с большим количеством параметров)

2. Для каждой из 4-х моделей будут использоваться одни и те же 30 эссе для валидации их работы и один и тот же промт, который совпадает с нашим Baseline (пункт 1). При этом метрики качества также будут совпадать с пунктом 1 и рассчитываться как по общей оценке, так и по 4-м компонентам

3. В этом блоке также были добавлены следующие характеристики моделей:
- Время загрузки модели (в сек.)
- Память, которую занимает загруженная модель (в МБ)
- Время оценки одного эссе (в сек.)
- Время оценки 30 эссе (в сек.)

4. В этом блоке есть 2 ноутбука папке Promting (в папке Model_comparison). В первом ноуте models_comparison.ipynb есть сравнение 3-х первых моделей. Из-за ограничений по GPU финальное сравнение происходит в ноуте Qwen_7B_models_comparison.ipynb (там же обучается последняя модель). Также 2 финальные таблицы со сравнением моделей находятся в этой же папке: models_time_df_new.csv содержить информацию о скорости работы моделей, df_all_metrics_new.csv содержит информацию о метриках качества

5. Из-за скорости работы и метрик была выбрана модель unsloth/Llama-3.2-3B-Instruct-bnb-4bit. У нее самое высокое значение QWK по сравнению с другими моделями, также классические метрики качества (MAE, MSE) практически не отличаются у Llama и легкого Qwen
   

## 4. Fine-Tuning c помощью LoRA
1. Использовали ту же модель и промт, что и в baseline. За исключением того, что для промта не указывали примеры. Плюс добавили учёт самого задания, для более корректной оценки TR
2. Настроили LoRA адаптер на слоях attention и MLP. Обучается на 1% параметров модели
3. Токенизировали текст с маскировкой промта, чтобы loss считался только по ответу. Отфильтровали примеры длинее 1024 токенов, их было порядка 5% от датасета
4. Для обучения взяли 2 эпохи, батч размера 16. Используется 8-bit оптимизатор и fp16 для экономии памяти. После каждой эпохи считается eval_loss на тестовой выборке, в конце загружается минимальный eval_loss.
